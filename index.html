About

Hi, my name is Sam.

I'm a self taught front-end developer, a University of Nottingham graduate and a Birmingham native.



I write single page web applications and websites using modern Javascript technologies alongside CSS and HTML.


If you want to know more about my skills or projects then scroll down below!

Skills

Vanilla JavaScript

I write a lot of code in vanilla Javascript and have spent a lot of time learning it. Although React and other frameworks make life easy, I believe in understanding how things work on a fundamental level.

React

I write single page web applications in React using the Context API for state management in smaller applications and Redux for larger applications. It's easy, it's fun and I love it.

Redux

Whenever I write a single page application that gets too big for React's Context API, I normally use Redux for state management.

Bootstrap

Bootstrap is my go-to CSS framework, I'm very well-versed with it and when I'm making something serious, I normally use it.

Materialize

Materialize is fun, I'm comfortable with it and although I haven't used it yet for any of my main projects, I am hoping to in the near future.

Sass

Sass is a blessing. I have used Less before but primarily I write in Sass simply because I discovered it first. I use both Koala and the Live Sass Compiler in VS Code for compilation.

HTML-5

Like most Web Developers, I use the latest standard of the HTML mark-up language to create every single app and webpage that I have created.

jQuery

Although it is old and it is going out of fashion a little bit, it's powerful and it saves re-inventing the wheel. I use jQuery primarily when I design websites (including this one!)

Git

I version control all my projects from the command line and everything is available for viewing on my github page (www.github.com/samjameshambridge)

npm

I included npm in this list because it's largely irreplaceable in a lot of applications, I've used it a lot and it's made my life a lot easier.
